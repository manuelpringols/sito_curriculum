Backend: Realizzazione con Spring Boot

1. Introduzione

Il backend del mio sito web è stato sviluppato utilizzando Spring Boot, un framework Java che permette di creare API RESTful scalabili e sicure. Ho implementato una serie di funzionalità per la gestione degli utenti e dei dati tramite un database collegato.

2. Autenticazione con JWT

Per garantire la sicurezza dell'applicazione, ho implementato l'autenticazione tramite JSON Web Token (JWT). Un esempio pratico di Spring Security richiede l'autenticazione con JWT. È possibile registrarsi per ottenere il token JWT e successivamente utilizzare questo token per effettuare il login.

3. Registrazione e Accesso

Registrazione

Inserire Email e password casuali

Accesso

Inserire Email e password scelti precedentemente

4. Connessione al Database

Il backend è collegato a un database MySQL. La configurazione di base è gestita tramite il file application.properties: 


spring.datasource.url=jdbc:postgresql://localhost:5432/sito-curriculum
spring.datasource.username=root
spring.datasource.password=password
spring.jpa.hibernate.ddl-auto=update
spring.jpa.properties.hibernate.default_schema=public
spring.jpa.show-sql=true

Processo di Autenticazione con JWT

1. Registrazione e Ricezione del Token JWT

Quando un utente si registra inserendo email e password, il backend genera un token JWT. Questo token viene restituito e salvato localmente nel browser dell'utente.

2. Autenticazione con JWT

Dopo la registrazione, l'utente accede utilizzando le stesse credenziali. Il token JWT salvato viene inviato con la richiesta di accesso. Se il token è valido, l'utente ottiene accesso a funzionalità specifiche protette.

3. Visualizzazione dei Progetti

I progetti sono accessibili a tutti gli utenti, indipendentemente dall'autenticazione. Tuttavia, l'accesso alle funzionalità protette avviene solo dopo che l'utente ha effettuato il login e ha fornito il token JWT.

Flusso del token JWT: Registrazione ➔ Ricezione JWT ➔ Salvataggio in localStorage

Disclaimer: Un Sito, Molte Possibilità 🚀

Questo sito web è stato deployato con GitHub Pages, il che significa che attualmente funziona come una pagina web statica. GitHub Pages è un modo rapido e gratuito per ospitare progetti come questo, ma c’è un dettaglio importante: il backend non è attualmente connesso.

Tuttavia, non lasciarti ingannare dalla sua semplicità! Tutto è già stato predisposto per essere integrato con un backend reale. Un server come Hetzner o qualsiasi altro hosting è perfetto per connettere il backend Spring Boot al frontend di questo sito.

Come fare?

  • Configura il server su Hetzner (o un altro provider) con Java e MySQL/PostgreSQL.
  • Lancia l'applicazione Spring Boot per gestire le API, l'autenticazione JWT e le operazioni del database.
  • Collega il frontend al backend, aggiornando semplicemente l'URL delle API nel progetto Angular.

Et voilà! 🔗 Il sito diventerà completamente dinamico, e tutto il processo di autenticazione, gestione dei dati e delle richieste sarà effettivamente funzionante come descritto!

Nel frattempo, puoi esplorare liberamente il frontend per vedere come sono strutturate le funzionalità e le logiche di un'applicazione moderna.

Vuoi vederlo in azione su un server vero? 🚀 Basta un piccolo deploy su Hetzner e il gioco è fatto!
